1 Anmeldung

Der Zugang zum Admin-Panel erfolgt ausschließlich über den Browser. Es wird kein zusätzliches Programm benötigt.

1.1 Schritt-für-Schritt-Anmeldung

  1. Browser öffnen und https://prasco.zapto.org/admin aufrufen.
  2. E-Mail-Adresse in das erste Eingabefeld eingeben.
  3. Passwort in das zweite Eingabefeld eingeben.
  4. Optional: Häkchen bei Angemeldet bleiben setzen, um die Sitzung auch nach dem Browser-Neustart aktiv zu halten.
  5. Auf den Button Anmelden klicken.
  6. Nach erfolgreicher Anmeldung erscheint automatisch das Dashboard.
Abb. 1.1 Anmeldeseite des Admin-Panels
💡 HTTPS ist Pflicht alle Verbindungen sind TLS-verschlüsselt. Das Zertifikat wird automatisch über Let's Encrypt erneuert.

1.2 Rollen & Berechtigungen

Jeder Benutzer hat genau eine Rolle. Die Rolle bestimmt, welche Bereiche des Admin-Panels sichtbar und bearbeitbar sind.

1.3 Passwort ändern / zurücksetzen

Eigenes Passwort ändern

  1. Oben rechts auf das Benutzer-Symbol klicken.
  2. Profil oder Passwort ändern auswählen.
  3. Aktuelles Passwort und zweimal das neue Passwort eingeben.
  4. Auf Speichern klicken.

Passwort eines anderen Benutzers zurücksetzen (Admin)

  1. Im Menü auf Benutzer navigieren.
  2. Den gewünschten Benutzer in der Liste suchen und auf Bearbeiten klicken.
  3. Im Feld Neues Passwort ein temporäres Passwort eingeben (mindestens 8 Zeichen).
  4. Auf Speichern klicken und das temporäre Passwort dem Benutzer sicher übermitteln.
⚠ Super-Admin-Konten können nur von anderen Super-Admins zurückgesetzt werden.

3 Beiträge verwalten

Der Bereich Beiträge ist das Herzstück des Systems. Hier werden alle Inhalte erstellt, bearbeitet und geplant, die auf den Displays angezeigt werden.

3.1 Beitrags-Übersicht

Die Tabelle zeigt alle Beiträge mit Titel, Kategorie, Status, Zeitraum und Displays. Über die Suche und den Kategorie-Filter lassen sich Beiträge schnell finden.

Abb. 3.1 Beitrags-Übersicht mit Such- und Filterfunktion

3.2 Neuen Beitrag erstellen

  1. Auf den Button + Neuer Beitrag klicken.
  2. Titel eingeben (Pflichtfeld, wird auf dem Display angezeigt).
  3. Inhalt eingeben. Mehrzeiliger Text und einfaches Markdown sind möglich.
  4. Kategorie aus der Dropdown-Liste wählen.
  5. Anzeigedauer in Sekunden eingeben (wie lange der Beitrag auf dem Display verweilt).
  6. Optional: Startdatum / Enddatum setzen, um die Sichtbarkeit zeitlich zu begrenzen.
  7. Optional: Eines oder mehrere Bilder hochladen oder eine YouTube-URL eintragen.
  8. Displays auswählen, auf denen der Beitrag erscheinen soll.
  9. Auf Speichern klicken.

3.3 Beitrag bearbeiten

In der Beitragsliste auf das Stift-Symbol ✎ klicken. Das Formular öffnet sich mit den aktuellen Daten vorausgefüllt. Änderungen mit Speichern bestätigen.

3.4 Beitrag löschen

Auf das Papierkorb-Symbol 🗑 klicken. Eine Sicherheitsabfrage erscheint. Mit Bestätigen wird der Beitrag unwiderruflich gelöscht.

⚠ Gelöschte Beiträge können nicht wiederhergestellt werden. Sicherungskopien sind ratsam.

3.5 Bilder & Medien

TypUnterstütztHinweis

3.6 Zeitplanung

Mit Startdatum und Enddatum lässt sich ein Beitrag exakt terminieren. Beiträge außerhalb des gültigen Zeitrahmens werden auf dem Display nicht gezeigt, bleiben aber im System erhalten.

💡 Für Aushänge, die immer gelten sollen, beide Datumsfelder leer lassen.
4 Kategorien

Kategorien gruppieren Beiträge thematisch. Jeder Beitrag gehört genau einer Kategorie an. Kategorien können auch Display-spezifisch gefiltert werden.

4.1 Kategorie erstellen

  1. Im Menü auf Kategorien klicken.
  2. Auf + Neue Kategorie klicken.
  3. Einen aussagekräftigen Namen eingeben (z. B. Veranstaltungen" oder Speiseplan").
  4. Optional: Farbe für die visuelle Unterscheidung auswählen.
  5. Auf Speichern klicken.

4.2 Kategorie bearbeiten / löschen

In der Kategorieliste das Stift-Symbol ✎ zum Bearbeiten oder das Papierkorb-Symbol 🗑 zum Löschen anklicken.

⚠ Eine Kategorie kann nicht gelöscht werden, solange ihr noch Beiträge zugeordnet sind. Beiträge zuerst in eine andere Kategorie verschieben oder löschen.
5 Displays

Im Bereich Displays werden alle angeschlossenen Bildschirme verwaltet. Jedes Display hat eine eigene Konfiguration, URL und Playlist.

5.1 Display hinzufügen

  1. Auf + Neues Display klicken.
  2. Einen eindeutigen Namen vergeben (z. B. Foyer EG" oder Kantine").
  3. Standort und optionale Beschreibung eingeben.
  4. Standard-Anzeigedauer festlegen (Sekunden pro Beitrag, wenn kein individueller Wert gesetzt).
  5. Optional: Kategoriefilter aktivieren, um nur bestimmte Kategorien anzuzeigen.
  6. Auf Speichern klicken. Die Display-URL wird automatisch generiert.

5.2 Display-URL & Kiosk-Modus

Jedes Display erhält eine eigene URL nach dem Schema https://prasco.zapto.org/display/<id>. Diese URL wird im Browser des Display-Geräts im Vollbild-Kiosk-Modus geöffnet.

💡 Android-TV-Geräte: Die PRASCO-App öffnet diese URL automatisch im Kiosk-Modus. Siehe Kapitel zur Android-App.

6 Benutzer & Rollen

Der Bereich Benutzer ist nur für Admins und Super-Admins zugänglich. Hier werden alle Konten verwaltet und Rollen zugewiesen.

6.1 Neuen Benutzer anlegen

  1. Auf + Neuer Benutzer klicken.
  2. Name und E-Mail-Adresse des Benutzers eingeben.
  3. Ein temporäres Passwort vergeben (mindestens 8 Zeichen).
  4. Die gewünschte Rolle aus der Dropdown-Liste wählen.
  5. Auf Speichern klicken und das temporäre Passwort dem neuen Benutzer mitteilen.

6.2 Rollen im Detail

RolleBeschreibung
BetrachterKann alle Beiträge und Displays einsehen, aber nicht ändern. Keine Schreibrechte.
EditorKann Beiträge erstellen und eigene Beiträge bearbeiten. Kein Zugriff auf Systemeinstellungen.
AdminVollzugriff auf Beiträge, Kategorien und Displays. Kann normale Benutzer und Editoren verwalten, aber keine anderen Admins.
Super-AdminVollzugriff auf alle Bereiche, einschließlich SSO-Konfiguration und Server-Einstellungen. Kann alle Rollen vergeben.

6.3 SSO (Single Sign-On)

Das System unterstützt SSO via SAML 2.0 (z. B. Microsoft Entra ID / Azure AD). Die Konfiguration ist unter Einstellungen SSO nur für Super-Admins zugänglich. Ausführliche Anleitung: Kapitel 8.3.

7 Live-Daten

Das System kann Echtzeit-Daten von externen Quellen einbinden und auf den Displays als Overlay oder eigener Beitrag darstellen.

7.1 ÖPNV-Abfahrten

Abfahrtsinformationen von Bussen und Bahnen werden über eine HAFAS- oder GTFS-Schnittstelle abgerufen.

  1. Unter Einstellungen Live-Daten ÖPNV die Haltestellen-ID oder den Haltestellenname eingeben.
  2. Die Anzahl der angezeigten Abfahrten festlegen.
  3. Optional: Linienfilter, um nur bestimmte Linien anzuzeigen.
  4. Auf Speichern klicken. Das Widget aktualisiert sich automatisch alle 60 Sekunden.
💡 Für westfälische Haltestellen empfiehlt sich die VRR-API (Verkehrsverbund Rhein-Ruhr). Die Haltestellen-IDs finden Sie in der VRR-Dokumentation.

7.2 Wetter

Das Wetter-Widget zeigt aktuelle Temperatur, Wetterlage und eine 3-Tage-Vorschau.

  1. Unter Einstellungen Live-Daten Wetter den Ort oder die Koordinaten eingeben.
  2. Optional: OpenWeatherMap API-Key eintragen (kostenlos registrierbar).
  3. Auf Speichern klicken.

7.3 Nachrichten & RSS

Beliebige RSS-Feeds können als Laufschrift oder Kachelansicht eingebunden werden.

  1. Unter Einstellungen Live-Daten RSS die URL des RSS-Feeds eingeben.
  2. Anzahl der angezeigten Meldungen festlegen.
  3. Auf Speichern klicken. Aktualisierungsintervall: 15 Minuten.

7.4 Verkehrslage

Aktuelle Verkehrsmeldungen für definierte Strecken werden über eine Verkehrs-API abgerufen. Konfiguration analog zu Wetter.

Hier lassen sich globale Standardwerte festlegen, die gelten, wenn ein Display keine eigene Konfiguration hat.

8.3 SSO-Konfiguration (Super-Admin)

Das System unterstützt SAML 2.0 für die Anmeldung per Microsoft 365 / Entra ID. Vorausgesetzt wird eine Enterprise-App-Registrierung im Azure-Portal.

  1. Im Azure-Portal eine neue Unternehmensanwendung anlegen und SAML als SSO-Methode wählen.
  2. Entitäts-ID und Antwort-URL (ACS-URL) aus dem Admin-Panel kopieren und in Azure eintragen.
  3. SAML-Zertifikat (Base64) aus Azure herunterladen und im Admin-Panel unter Einstellungen SSO Zertifikat einfügen.
  4. Anmelde-URL und Azure AD-ID aus Azure kopieren und eintragen.
  5. Auf Speichern & Testen klicken.
💡 Eine vollständige Schritt-für-Schritt-Anleitung mit Screenshots finden Sie in der Datei SSO-SETUP.md im Projektverzeichnis.

8.4 Backup & Wiederherstellung

Das System erstellt automatisch täglich ein Datenbank-Backup. Manuelle Backups können jederzeit über den Server-Zugang ausgeführt werden:

bash /opt/Prasco/pg_backup.sh

Zur Wiederherstellung:

psql -U prasco prasco < /opt/backups/prasco_DATUM.sql
