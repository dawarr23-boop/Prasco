1 Anmeldung

Der Zugang zum Admin-Panel erfolgt ausschließlich über den Browser. Es wird kein zusätzliches Programm benötigt.

1.1 Schritt-für-Schritt-Anmeldung

  1. Browser öffnen und https://prasco.zapto.org/admin aufrufen.
  2. E-Mail-Adresse in das erste Eingabefeld eingeben.
  3. Passwort in das zweite Eingabefeld eingeben.
  4. Optional: Häkchen bei Angemeldet bleiben setzen, um die Sitzung auch nach dem Browser-Neustart aktiv zu halten.
  5. Auf den Button Anmelden klicken.
  6. Nach erfolgreicher Anmeldung erscheint automatisch das Dashboard.
Abb. 1.1 Anmeldeseite des Admin-Panels
💡 HTTPS ist pflicht alle Verbindungen sind TLS-verschlüsselt. Das Zertifikat wird automatisch über Let's Encrypt erneuert.

1.2 Rollen & Berechtigungen

Jeder Benutzer hat genau eine Rolle. Die Rolle bestimmt, welche Bereiche des Admin-Panels sichtbar und bearbeitbar sind.

1.3 Passwort ändern / zurücksetzen

Eigenes Passwort ändern

  1. Oben rechts auf das Benutzer-Symbol klicken.
  2. Profil oder Passwort ändern auswählen.
  3. Aktuelles Passwort und zweimal das neue Passwort eingeben.
  4. Auf Speichern klicken.

Passwort eines anderen Benutzers zurücksetzen (Admin)

  1. Im Menü auf Benutzer navigieren.
  2. Den gewünschten Benutzer in der Liste suchen und auf Bearbeiten klicken.
  3. Im Feld Neues Passwort ein temporäres Passwort eingeben (mindestens 8 Zeichen).
  4. Auf Speichern klicken und das temporäre Passwort dem Benutzer sicher übermitteln.
⚠ Super-Admin-Konten können nur von anderen Super-Admins zurückgesetzt werden.

3 Beiträge verwalten

Beiträge sind die zentralen Inhaltseinheiten in IT-Westfalen Signature Screen. Jeder Beitrag wird auf einem oder mehreren Displays in einer definierten Reihenfolge abgespielt.

Abb. 3.1 Beiträge-Übersicht mit Listenansicht, Filter und Aktionsbuttons

3.2 Beitrag erstellen Schritt für Schritt

  1. Im Menü auf Beiträge klicken.
  2. Oben rechts auf + Neuer Beitrag klicken.
  3. Den gewünschten Beitrags-Typ aus der Auswahlliste wählen.
  4. Titel des Beitrags eingeben.
  5. Je nach Typ: Text eingeben, Datei hochladen oder URL einfügen.
  6. Eine Kategorie aus der Dropdown-Liste zuweisen.
  7. Das Ziel-Display auswählen (Mehrfachauswahl möglich).
  8. Optional: Anzeigedauer, Start- und Endzeit festlegen.
  9. Den Schalter Beitrag aktiv auf EIN stellen.
  10. Auf Speichern klicken der Beitrag erscheint sofort auf dem Display.

3.4 Zeitsteuerung (Scheduling)

Beiträge können auf bestimmte Zeiträume beschränkt werden ideal für saisonale Inhalte, Veranstaltungen oder Wochenend-Angebote.

Startzeit
Datum und Uhrzeit ab wann der Beitrag erscheint (leer = sofort aktiv)
Endzeit
Datum und Uhrzeit bis wann der Beitrag erscheint (leer = kein Ablaufdatum)

Beispiele

SzenarioStartzeitEndzeit
Weihnachts-Aktion01.12. 00:0026.12. 23:59
Tagesangebot (tägl. wiederholt)Heute 11:00Heute 14:00 (tägl. manuell anpassen)

💡 Ein Beitrag mit Endzeit in der Vergangenheit wird nicht angezeigt, bleibt aber gespeichert und kann mit neuer Endzeit reaktiviert werden.

5 Displays

Abb. 5.1 Displays-Übersicht mit Verbindungsstatus und letztem Ping

5.1 Display anlegen

  1. Im Menü auf Displays klicken.
  2. Auf + Neues Display klicken.
  3. Einen sprechenden Namen eingeben (z. B. Empfang EG").
  4. Einen eindeutigen Bezeichner eingeben nur Kleinbuchstaben und Bindestriche (z. B. empfang-eg).

5.2 Display-URL & Kiosk-Modus

Die URL eines Displays lautet:

https://prasco.zapto.org/display/<bezeichner>

Für einen dauerhaften Vollbild-Kiosk-Modus unter Chromium (z. B. Raspberry Pi) empfiehlt sich folgender Startbefehl:

chromium-browser --kiosk --noerrdialogs --disable-infobars \
  --incognito --check-for-update-interval=604800 \
  "https://prasco.zapto.org/display/<bezeichner>"
💡 Nach dem Speichern des Displays erscheint in der Übersichtsliste der Link Display öffnen damit kann man die Vollbildansicht direkt im Browser testen, ohne den Kiosk-Modus einzurichten.

7 Live-Daten (ÖPNV, Verkehr, Wetter & Nachrichten)

IT-Westfalen Signature Screen kann Echtzeit-Informationen automatisch zwischen regulären Beiträgen einblenden. Die Konfiguration erfolgt im Bereich Live-Daten.

Abb. 7.1 Live-Daten Übersicht mit vier Konfigurationskarten

7.1 ÖPNV-Abfahrten

Abb. 7.2 ÖPNV-Konfiguration mit Stationssuche und Fahrzeugfiltern
  1. Schalter ÖPNV-Abfahrten aktivieren auf EIN stellen.
  2. Im Suchfeld eine Haltestelle eingeben (z. B. Ahlen Bahnhof") und kurz warten.
  3. Die gewünschte Haltestelle aus den Suchergebnissen auswählen.
  4. Zeitfenster einstellen: Wie viele Minuten in die Zukunft sollen Abfahrten gezeigt werden? (Standard: 60 min)
  5. Maximale Anzahl Abfahrten festlegen (Standard: 10).
  6. Nicht benötigte Verkehrsmittel abwählen (Bus, S-Bahn, U-Bahn, Tram, Regional, Fern, Fähre).
  7. Unter Auf Displays anzeigen: Ziel-Displays auswählen (Mehrfachauswahl).
  8. Auf Speichern klicken das Widget erscheint ab dem nächsten Live-Daten-Intervall.
💡 Die Daten kommen über die freie HAFAS-API. Es ist kein eigener API-Key erforderlich.

7.2 Autobahn-Verkehrslage

  1. Schalter Verkehrslage aktivieren auf EIN stellen.
  2. Preset-Standort wählen (z. B. Ahlen, Münster, Dortmund) oder manuell Autobahnen aus der Liste aktivieren.
  3. Meldungstypen auswählen: Warnungen, Baustellen, Sperrungen.
  4. Ziel-Displays auswählen.
  5. Auf Speichern klicken.

7.3 Wetter & Vorhersage

  1. Schalter Wetter aktivieren auf EIN stellen.
  2. Ort in das Suchfeld eingeben (z. B. Ahlen, NRW") und Enter drücken.
  3. Ergebnis aus der Auswahlliste auswählen Koordinaten werden automatisch gespeichert.
  4. Anzahl Vorhersage-Tage einstellen (17 Tage; 1 = nur heute).
  5. Ziel-Displays auswählen.
  6. Auf Speichern klicken.
✓ Daten kommen von Open-Meteo komplett kostenlos, kein API-Key erforderlich, DSGVO-konform.

7.4 Nachrichten-Feed

  1. Schalter Nachrichten aktivieren auf EIN stellen.
  2. RSS-Feed-URL eingeben (z. B. https://www.tagesschau.de/xml/rss2).
  3. Maximale Anzahl Nachrichten einstellen.
  4. Auf Speichern klicken.

8.3 KI-Assistent einrichten

  1. OpenAI-Konto unter platform.openai.com erstellen (falls noch nicht vorhanden).
  2. Unter API Keys einen neuen Key erstellen und kopieren.
  3. Im Admin-Panel: Einstellungen KI-Assistent öffnen.
  4. Den API-Key in das Feld OpenAI API-Key einfügen.
  5. Gewünschtes Modell auswählen (Standard: gpt-4o-mini).
  6. Auf Speichern klicken.
  7. Beim nächsten Öffnen eines Beitrags erscheint der KI-Button neben dem Textfeld.
⚠ Der API-Key wird verschlüsselt in der Datenbank gespeichert, aber nie im Browser-Quellcode sichtbar gemacht. OpenAI verrechnet Tokens; die Kosten liegen bei normaler Nutzung unter 1 € / Monat.

8.4 SSO / LDAP konfigurieren (Super-Admin)

IT-Westfalen Signature Screen unterstützt Single Sign-On (SSO) über zwei Methoden:

Azure AD / Microsoft Entra ID (OAuth2/OIDC)

  1. Im Azure-Portal eine App-Registrierung erstellen.
  2. Redirect-URI eintragen: https://prasco.zapto.org/auth/azure/callback
  3. Client-ID und Client-Secret notieren.
  4. Im Admin-Panel: Einstellungen SSO Azure AD öffnen.
  5. Tenant-ID, Client-ID und Client-Secret eintragen.
  6. Auf Speichern & Verbindung testen klicken.

LDAP / Active Directory

  1. LDAP-Server-URL eingeben (z. B. ldap://dc01.firma.local:389).
  2. Bind-DN und Bind-Passwort eintragen (Dienstkonto mit Lesezugriff).
  3. Basis-DN für die Benutzersuche angeben (z. B. OU=Users,DC=firma,DC=local).
  4. Attribut-Mapping konfigurieren (E-Mail, Anzeigename).
  5. Auf Speichern & Verbindung testen klicken.
💡 Eine ausführliche Anleitung mit Screenshots für beide Verfahren ist in der Datei SSO-SETUP.md auf dem Server dokumentiert.

9.2 Server-Logs einsehen

Für tiefergehende Diagnosen können die Container-Logs auf dem Server abgerufen werden (SSH-Zugang erforderlich):

# Letzte 100 Zeilen des App-Logs
docker logs --tail 100 prasco-app

# Logs in Echtzeit verfolgen (Strg+C zum Beenden)
docker logs -f prasco-app

# Nur Fehler anzeigen
docker logs prasco-app 2>&1 | grep -i "error\|fatal"

9.3 System neu starten

  1. Per SSH auf den Server verbinden: ssh root@212.227.20.158
  2. In das Projektverzeichnis wechseln: cd /opt/Prasco
  3. App-Container neu starten: docker compose -f docker-compose.prod.yml restart app
  4. Status prüfen: docker compose -f docker-compose.prod.yml ps
  5. Im Browser https://prasco.zapto.org/api/health aufrufen bei Erfolg erscheint {"status":"ok"}.
⚠ Ein vollständiger Neustart aller Container (docker compose down && docker compose up -d) beendet kurzzeitig alle laufenden Displays.
